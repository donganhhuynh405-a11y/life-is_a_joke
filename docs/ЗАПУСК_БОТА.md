# 🤖 Полная инструкция по запуску Trading Bot

> **Проблема:** Бот не запускается из-за нехватки места на диске при установке `torch` (~916 МБ),
> `tensorflow-cpu` (~274 МБ) и `nvidia_cublas_cu12` (~594 МБ) из PyPI.
>
> **Решение:** Тяжёлые ML-пакеты вынесены в отдельный файл `requirements-ml.txt`.
> Основной `requirements.txt` содержит только лёгкие зависимости (≈ 300 МБ),
> бот запускается и работает без PyTorch и TensorFlow благодаря lazy-импортам.

## 1. Быстрый старт

Если бот уже установлен, но не запускается из-за нехватки диска:

```bash
# 1. Освободить место на диске (запускать от root)
sudo bash /opt/trading-bot/scripts/free_disk_space.sh

# 2. Обновить код (чтобы получить исправленный requirements.txt)
cd /opt/trading-bot
sudo git pull origin main

# 3. Удалить старый venv (в котором могут быть оставшиеся тяжёлые пакеты)
sudo rm -rf /opt/trading-bot/venv

# 4. Запустить бота — venv_prestart.sh пересоздаст venv с лёгкими зависимостями
sudo systemctl start trading-bot
sudo journalctl -u trading-bot -f
```

## 2. Свежая установка с нуля

### 2.1. Системные требования

| Параметр | Минимум | Рекомендуется |
|----------|---------|---------------|
| ОС | Ubuntu 20.04 / Debian 11 | Ubuntu 22.04 LTS |
| CPU | 1 ядро | 2+ ядра |
| RAM | 512 МБ | 1 ГБ+ |
| Диск | 2 ГБ свободно | 5 ГБ+ (10 ГБ если нужен ML) |
| Python | 3.10 | 3.11 / 3.12 |
| PostgreSQL | 13+ | 15+ |

### 2.2. Установка системных зависимостей

```bash
sudo apt-get update
sudo apt-get install -y \
    python3 python3-pip python3-venv \
    git curl wget \
    build-essential libssl-dev libffi-dev python3-dev
```

### 2.3. Создание пользователя сервиса

```bash
sudo useradd --system --shell /bin/false --home /opt/trading-bot tradingbot
sudo mkdir -p /opt/trading-bot
sudo chown tradingbot:tradingbot /opt/trading-bot
```

### 2.4. Клонирование репозитория

```bash
sudo git clone https://github.com/donganhhuynh405-a11y/life-is_a_joke.git /opt/trading-bot
sudo chown -R tradingbot:tradingbot /opt/trading-bot
```

### 2.5. Создание виртуального окружения и установка зависимостей

```bash
cd /opt/trading-bot

# Создать venv
python3 -m venv venv

# Установить ТОЛЬКО лёгкие зависимости (не требует torch/tensorflow)
venv/bin/pip install --upgrade pip
venv/bin/pip install -r requirements.txt

# Передать владение сервисному пользователю
sudo chown -R tradingbot:tradingbot /opt/trading-bot/venv
```

### 2.6. Настройка конфигурации

```bash
# Скопировать шаблон .env
sudo cp /opt/trading-bot/.env.template /opt/trading-bot/.env

# Настроить ключи API и прочие параметры
sudo nano /opt/trading-bot/.env
```

Обязательные параметры в `.env`:

```env
# Биржа
BINANCE_API_KEY=your_api_key_here
BINANCE_API_SECRET=your_api_secret_here
EXCHANGE_TESTNET=false         # true для тестовой сети

# База данных (PostgreSQL)
DB_HOST=localhost
DB_PORT=5432
DB_NAME=trading_bot
DB_USER=trading_bot_user
DB_PASSWORD=your_secure_password

# Telegram (опционально — уведомления)
TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_CHAT_ID=your_chat_id

# Торговля
TRADING_ENABLED=true
BASE_ASSET=USDT
QUOTE_ASSET=BTC
```

### 2.7. Настройка PostgreSQL

```bash
sudo apt-get install -y postgresql
sudo -u postgres psql -c "CREATE USER trading_bot_user WITH PASSWORD 'your_secure_password';"
sudo -u postgres psql -c "CREATE DATABASE trading_bot OWNER trading_bot_user;"
```

### 2.8. Установка systemd-сервиса

```bash
# Скопировать файл сервиса
sudo cp /opt/trading-bot/deployment/systemd/trading-bot.service \
        /etc/systemd/system/trading-bot.service

# Сделать скрипты запуска исполняемыми
sudo chmod +x /opt/trading-bot/scripts/venv_prestart.sh
sudo chmod +x /opt/trading-bot/scripts/start_bot.sh

# Перезагрузить systemd и включить автозапуск
sudo systemctl daemon-reload
sudo systemctl enable trading-bot

# Запустить бота
sudo systemctl start trading-bot

# Проверить статус
sudo systemctl status trading-bot
sudo journalctl -u trading-bot -f
```

## 3. Обновление существующей установки

```bash
cd /opt/trading-bot

# 1. Остановить бота
sudo systemctl stop trading-bot

# 2. Получить последние изменения
sudo git pull origin main

# 3. Если requirements.txt изменился — удалить старый stamp,

# 4. Запустить бота (venv_prestart.sh автоматически обновит пакеты)
sudo systemctl start trading-bot
sudo journalctl -u trading-bot -f
```

## 4. Полный функционал с ML и AI

Все ML/AI-стратегии работают через lazy-импорты: если PyTorch и TensorFlow не установлены —
бот просто не использует эти стратегии и продолжает работать в штатном режиме.

### 4.1. Что работает WITHOUT torch/tensorflow

| Функционал | Работает без ML? |
|------------|-----------------|
| Торговля на Binance / CCXT | ✅ |
| Технический анализ (RSI, MACD, BB, SMA…) | ✅ |
| Управление рисками | ✅ |
| PostgreSQL база данных | ✅ |
| Telegram-уведомления | ✅ |
| REST API (FastAPI) | ✅ |
| Мониторинг (Prometheus) | ✅ |
| Анализ новостей (RSS) | ✅ |
| Portfolio optimisation (scikit-learn) | ✅ |
| Продвинутые ML-стратегии (PPO, LSTM, трансформеры) | ❌ (нужен ML) |

### 4.2. Установка ML-пакетов (опционально, если ≥ 5 ГБ свободно)

```bash
# ВНИМАНИЕ: требует минимум 5 ГБ свободного места
# Проверить: df -h /opt
df -h /opt

# Установить ML-зависимости (CPU-only torch из официального CDN PyTorch)
cd /opt/trading-bot
venv/bin/pip install -r requirements-ml.txt

# После установки перезапустить бота
sudo systemctl restart trading-bot
```

> **Важно:** `requirements-ml.txt` устанавливает `torch` с официального CDN PyTorch
> (`https://download.pytorch.org/whl/cpu`). Это CPU-only wheel (~200 МБ),
> а не CUDA-версия с PyPI (~916 МБ + ~594 МБ nvidia_cublas_cu12).

### 4.3. Включение конкретных ML-стратегий в конфиге

После установки ML-пакетов настройте `config.yaml` или `.env`:

```env
# Включить PPO-стратегию
USE_PPO_STRATEGY=true

# Включить LSTM-предсказания
USE_LSTM_PREDICTIONS=true

# Включить Transformer sentiment анализ
USE_TRANSFORMER_SENTIMENT=true
```

## 5. Структура файлов зависимостей

```
requirements.txt        ← ОСНОВНОЙ файл (≈300 МБ)
                          Используется systemd-сервисом автоматически.
                          Без torch / tensorflow / CUDA.
                          Бот полностью работоспособен.

requirements-ml.txt     ← ОПЦИОНАЛЬНЫЙ файл (≈800 МБ дополнительно)
                          Устанавливайте вручную только при наличии
                          ≥ 5 ГБ свободного диска.
                          Содержит:
                            - tensorflow-cpu (≈274 МБ)
                            - torch CPU-only с CDN (≈200 МБ)
                            - transformers, optuna, tensorboard…

requirements-test.txt   ← Только для разработчиков / CI
```

## 7. Освобождение места на диске

### Автоматически (скрипт)

```bash
sudo bash /opt/trading-bot/scripts/free_disk_space.sh
```

### Вручную (шаг за шагом)

```bash
# Проверить текущее использование диска
df -h

# 1. Очистить журнал systemd (часто 500 МБ – 2 ГБ)
sudo journalctl --disk-usage
sudo journalctl --vacuum-size=100M

# 2. Очистить кэш apt
sudo apt-get clean

# 3. Очистить кэш pip
sudo pip3 cache purge
sudo rm -rf /root/.cache/pip

# 4. Удалить временные .whl файлы
sudo find /tmp -maxdepth 2 -name "*.whl" -delete
sudo find /tmp -maxdepth 2 -name "pip-*" -mtime +1 -exec rm -rf {} +

# 5. Удалить __pycache__
find /opt/trading-bot -type d -name "__pycache__" -exec rm -rf {} + 2>/dev/null || true

# 6. Если ML установлен и больше не нужен — удалить тяжёлые пакеты
/opt/trading-bot/venv/bin/pip uninstall -y torch tensorflow-cpu nvidia_cublas_cu12 \
    cuda_bindings cuda_pathfinder 2>/dev/null || true

# 7. Проверить после очистки
df -h

# 8. Перезапустить бота
sudo systemctl restart trading-bot
```

## 8. Диагностика и решение частых проблем

### Проблема: `status=203/EXEC` при старте сервиса

**Причина:** скрипт `venv_prestart.sh` не найден или не является исполняемым.

```bash
# Исправить права доступа
sudo chmod +x /opt/trading-bot/scripts/venv_prestart.sh
sudo chmod +x /opt/trading-bot/scripts/start_bot.sh

# Убедиться, что файл сервиса обновлён
sudo cp /opt/trading-bot/deployment/systemd/trading-bot.service \
        /etc/systemd/system/trading-bot.service
sudo systemctl daemon-reload
sudo systemctl restart trading-bot
```

### Проблема: `[Errno 28] No space left on device`

**Причина:** недостаточно места на диске при установке пакетов.

```bash
# Освободить место
sudo bash /opt/trading-bot/scripts/free_disk_space.sh

# Удалить старый venv (если в нём уже установлены тяжёлые пакеты)
sudo rm -rf /opt/trading-bot/venv

# Перезапустить — venv_prestart.sh пересоздаст его с лёгкими зависимостями
sudo systemctl start trading-bot
```

### Проблема: `ModuleNotFoundError: No module named 'sqlalchemy'`

**Причина:** venv существует, но пакеты не установлены (venv повреждён или requirements изменились).

```bash
# Принудительно переустановить зависимости
sudo rm -f /opt/trading-bot/venv/.requirements_installed
sudo systemctl restart trading-bot
```

### Проблема: бот запускается, но ML-стратегии не активны

```bash
# Проверить, установлен ли torch
/opt/trading-bot/venv/bin/python3 -c "import torch; print(torch.__version__)"

# Если нет — установить ML-зависимости (при наличии ≥ 5 ГБ)
df -h /opt
/opt/trading-bot/venv/bin/pip install -r /opt/trading-bot/requirements-ml.txt
sudo systemctl restart trading-bot
```

### Проблема: нет подключения к Telegram

```bash
# Проверить токен и chat_id
grep -E 'TELEGRAM' /opt/trading-bot/.env

# Тест подключения
/opt/trading-bot/venv/bin/python3 -c "
import requests, os
token = os.getenv('TELEGRAM_BOT_TOKEN', '')
r = requests.get(f'https://api.telegram.org/bot{token}/getMe')
print(r.json())
"
```

### Просмотр логов бота

```bash
# Журнал systemd (последние события)
sudo journalctl -u trading-bot -n 100 --no-pager

# Файлы логов бота
ls -lh /var/log/trading-bot/
tail -f /var/log/trading-bot/bot.log 2>/dev/null || \
    sudo journalctl -u trading-bot -f
```

## Итоговая схема работы автоматической установки

```
systemctl start trading-bot
       │
       ▼
ExecStartPre: chmod +x venv_prestart.sh   (root)
       │
       ▼
ExecStartPre: venv_prestart.sh            (root)
  ├── создаёт /opt/trading-bot/venv  (если отсутствует)
  ├── проверяет место на диске (нужно ≥ 512 МБ)
  ├── pip install -r requirements.txt  (ТОЛЬКО если хэш изменился)
  └── chown -R tradingbot:tradingbot venv
       │
       ▼
ExecStart: start_bot.sh                   (tradingbot user)
  └── exec venv/bin/python3 src/main.py
```
